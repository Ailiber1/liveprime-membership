PHASE 1 プロジェクト初期設定 1〜2日

Claude Codeでプロジェクトの骨格を一気に生成する。

01
Next.js プロジェクト生成 + 基本設定
CLAUDE.md作成 → npx create-next-app → 依存関係インストール
▾
CLAUDE CODE に指示する内容
# まずCLAUDE.mdを作成してプロジェクトルールを定義 Next.jsのApp Routerプロジェクトを作成して。 TypeScript, Tailwind CSS, ESLint を有効化。 Supabase, Stripe のSDKもインストール。 CLAUDE.mdにプロジェクト構成と命名規則を書いて。
CLAUDE.md package.json next.config.ts tailwind.config.ts
02
ディレクトリ構成の設計
app/ を機能別に分割、共通コンポーネント定義
▾
CLAUDE CODE に指示する内容
以下のディレクトリ構成でファイルを作成して: app/ ├── (public)/ # 未ログインページ │ ├── page.tsx # LP/トップ │ ├── login/ │ └── register/ ├── (member)/ # 会員専用エリア │ ├── dashboard/ │ ├── videos/ │ └── settings/ ├── (admin)/ # 管理画面 │ ├── users/ │ ├── billing/ │ └── content/ ├── api/ │ ├── webhooks/stripe/ │ └── auth/ lib/ ├── supabase/ ├── stripe/ └── utils/
03
環境変数 + Supabase初期セットアップ
.env.local設定、Supabaseプロジェクト作成、テーブル設計
▾
CLAUDE CODE に指示する内容
Supabaseのマイグレーションファイルを作成して。 テーブル設計: - profiles: ユーザープロフィール（user_id, display_name, plan, created_at） - subscriptions: サブスク状態（stripe_customer_id, status, current_period_end） - videos: 動画コンテンツ（title, url, thumbnail, is_premium, created_at） - watch_history: 視聴履歴 RLS（Row Level Security）ポリシーも設定して。 会員は自分のデータのみ、管理者は全データにアクセス可能に。
.env.local.example supabase/migrations/001_initial.sql

CLAUDE CODE に指示する内容
Supabase Authで認証機能を作って。 - メール+パスワード登録 - Google OAuth ログイン - middleware.tsでセッション検証 - (member)/ 配下は認証必須にリダイレクト - (admin)/ 配下はadminロール必須
middleware.ts lib/supabase/server.ts lib/supabase/client.ts
05
登録・ログインUI
フォームバリデーション、エラーハンドリング、レスポンシブ
▾
CLAUDE CODE に指示する内容
登録ページとログインページを作って。 - Tailwind CSSでモダンなデザイン - バリデーション（メール形式、パスワード8文字以上） - エラーメッセージの日本語表示 - Google OAuthボタン - 既存HPから遷移できるよう、共通ヘッダー/フッターのデザインを合わせて
app/(public)/login/page.tsx app/(public)/register/page.tsx components/AuthForm.tsx
PHASE 3 決済機能（Stripe） 3〜5日

サブスクリプション型月額課金の実装。ここが最も複雑。

06
Stripe商品・料金プラン設定
Stripeダッシュボードで商品作成 → APIキー取得
▾
手動作業（Stripe Dashboard）
# Stripe側で手動設定が必要 1. Stripeアカウント作成 → テストモード有効化 2. Products → 月額プラン作成（例: ¥980/月, ¥9,800/年） 3. API Keys → テストキー取得 4. Webhook endpointを登録: https://yourdomain.com/api/webhooks/stripe
07
Checkout Session + Webhook
決済フロー実装、Webhook受信でDB更新
▾
CLAUDE CODE に指示する内容
Stripe Checkoutでサブスクリプション決済を実装して。 1. POST /api/checkout → Stripe Checkout Sessionを生成 → 成功時にリダイレクトURL設定 2. POST /api/webhooks/stripe → checkout.session.completed → customer.subscription.updated → customer.subscription.deleted → 各イベントでSupabaseのsubscriptionsテーブル更新 3. Webhookの署名検証を必ず実装して。 4. カスタマーポータル（プラン変更・解約）のURLも /api/portal で生成して。
app/api/checkout/route.ts app/api/webhooks/stripe/route.ts app/api/portal/route.ts lib/stripe/index.ts
08
料金プラン選択UI + 決済導線
プランカード、決済ボタン、成功/失敗ページ
▾
CLAUDE CODE に指示する内容
料金プラン選択ページを作って。 - 月額プランと年額プランのカード表示 - 「今すぐ始める」ボタン → Stripe Checkout へ - 決済成功ページ（/member/welcome） - 決済キャンセルページ - 現在のプラン状態表示コンポーネント
🏁
中間マイルストーン：ここまでで「登録 → ログイン → 決済 → 会員ステータス反映」の一連のフローが動作確認可能。Stripeのテストカードで通しテストを行う。
PHASE 4 会員限定コンテンツ 3〜4日

動画コンテンツの閲覧制御とダッシュボード。

09
コンテンツ閲覧制御
プラン別アクセス制御、動画プレーヤー、署名付きURL
▾
CLAUDE CODE に指示する内容
会員限定コンテンツの閲覧制御を実装して。 - videosテーブルのis_premiumフラグで無料/有料を区別 - 有料コンテンツはサブスク有効会員のみ表示 - 動画URLは署名付きURL（Supabase Storage）で直リンク不可に - 未契約ユーザーにはプレビュー + 「プランに加入」CTAを表示 - Server Componentでサーバー側でアクセスチェック
app/(member)/videos/page.tsx app/(member)/videos/[id]/page.tsx components/VideoPlayer.tsx components/PaywallGate.tsx
10
会員ダッシュボード
マイページ、プラン情報、視聴履歴、プロフィール編集
▾
CLAUDE CODE に指示する内容
会員ダッシュボードを作って。 - 現在のプラン・次回更新日を表示 - 視聴履歴一覧（最近見た動画） - プロフィール編集（表示名、アバター） - プラン変更・解約ボタン → Stripe Customer Portal - パスワード変更機能

PHASE 6 セキュリティ・テスト・デプロイ 3〜4日

本番リリースに向けた堅牢化と最終調整。

13
セキュリティ強化
CSRF, XSS, レート制限, 個人情報管理
▾
CLAUDE CODE に指示する内容
セキュリティ対策を実装して。 - CSRFトークン検証 - XSS対策（サニタイズ処理） - APIエンドポイントにレート制限（upstash/ratelimit） - Stripeの署名検証の再確認 - Content Security Policy ヘッダー設定 - 個人情報のログ出力禁止 - npm audit で脆弱性チェック
14
E2Eテスト + デプロイ
Playwright テスト、Vercelデプロイ、既存HP連携
▾
CLAUDE CODE に指示する内容
Playwrightで主要フローのE2Eテストを書いて。 - 新規登録 → ログイン → 決済 → コンテンツ閲覧 - 未ログインで会員ページアクセス → リダイレクト確認 - 管理画面のアクセス制御テスト - Stripe Webhookのモックテスト その後、Vercelにデプロイ設定して。 環境変数をVercelにセット。 既存HPからのリンク遷移が正しく動くか確認用チェックリストも作って。
tests/e2e/auth.spec.ts tests/e2e/payment.spec.ts tests/e2e/admin.spec.ts vercel.json
🚀
リリース：テスト環境でクライアント確認 → Stripeを本番モードに切替 → 本番デプロイ → 既存HPにリンク設置 → 運用開始

⚠ Claude Codeだけでは完結しない作業

以下はClaude Codeの外で手動対応が必要です：
・Stripeアカウント作成、本人確認、商品登録（ダッシュボード操作）
・Supabaseプロジェクト作成、APIキー取得
・Vercelアカウント設定、ドメイン接続
・既存HPのHTML編集（リンク追加）
・中国決済（WeChat Pay / Alipay）は別途Stripe Connect等の審査が必要
・クライアントとの仕様確認・デザインすり合わせ
